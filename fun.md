# Функциональные требования SCADA:

## 1. Объектная модель и визуализация HMI

1. Поддержка faceplate (лицевой панели) объектов и групп, возможность отображения во всплывающем окне 
2. Наследование объектов 
3. Группировка объектов   
4. Объектная модель базы каналов, привязка тегов к объектам по шаблону 
5. История объекта (проигрыватель на основе протокола) 
6. Поддержка векторной графики  
7. Расширяемая библиотека визуальных элементов управления, трендов, таблиц 
8. Возможность сохранения текущей версии компонента из библиотеки прямо в проекте 
9. Компоновка и размещение классов проекта, трассировка связей между объектами на карте, генеративный дизайн 
10. Вкладки в окне оператора (мульти-оконность), загрузка окон при подключении пользователя к системе из настроек конфигурации 
11. Копирование и вставка карт и объектов между проектами 
12. Мобильный и адаптивный web‑клиент, поддержка push-уведомлений
13. Генерация всего проекта с помощью ИИ на основе шаблонов и ранее разработанных проектов 

## 2. Программирование и логика исполнения

1. Возможность вносить изменения в контроллер во время его работы, без перезагрузки 
2. Возможность вносить изменения в работающий проект SCADA без перезагрузки 
3. Поддержка скриптов для бизнес-логики (lua, java, js, python, jexl) 
4. Поддержка создания и выполнения процедурных функциональных диаграмм согласно стандарту ISA-88 
5. Встроенный отладчик скриптов 
6. Разработка дополнительных сценариев работы с помощью макросов, запуск по событиям или по расписанию 
7. Возможность генерации кода по текстовому описанию с помощью ИИ 
8. Выполнение проверок входных данных на стороне веб-сервера перед отправкой запроса контроллеру 

## 3. Инструменты разработки и UX редакторов

1. Единая среда коллаборативной разработки проектов, включая работу с контроллером 
2. Поиск в инспекторе объектов по имени объекта и свойству, глобальный поиск по проекту 
3. Возможность отмены действий в редакторе 
4. Редактирование функций проекта и обработчиков в одном общем редакторе кода. В текущем варианте код для редактирования нужно открыть в конкретном обработчике и только там сохранить. 
5. Поддержка горячих клавиш в редакторе и в мониторе 

## 4. Протоколы и драйверы

1. Поддержка протоколов работы с контроллером: OPC-UA, Modbus, Snap7, Profinet, MQTT 
2. Встроенный OPC-UA сервер для работы со сторонними системами 
3. Эмуляторы протоколов, тегов, объектов и устройств для тестирования без физического оборудования

## 5. Интеграции предприятия и межсерверное взаимодействие

1. Импорт данных с EPLAN, добавление в EPLAN элементов управления 
2. Межсерверное взаимодействие по объектам, тегам 
3. Поддержка работы через API с ERP системами 
4. REST API / WebSocket API для доступа к данным и для управления

## 6. Данные, архивирование и аналитика

1. Журнал событий (условия тревог, приоритеты, уведомления, снятие/подтверждение)
2. Система архивирования 
3. Журнал аварий, система обработки ошибок 
4. Генератор отчетов (периодические и по‑требованию (PDF, CSV, Excel), шаблоны отчетов, рассылка)

## 7. Производственные функции (рецепты и прослеживаемость)

1. Управление рецептами (параметры, шаги техпроцесса), загрузка рецептов в контроллер 
2. Прослеживаемость по техпроцессам как в режиме онлайн, так и на основе архивных данных 

## 8. Надёжность, развертывание и окружения

1. Кроссплатформенность решения 
2. Поддержка работы с серверами для производства и для разработки 
3. Резервирование серверов, автоматическое переключение с основного сервера на резервный и обратно, как при работе с контроллерами, так и при работе с интерфейсом пользователей. Диагностика отключений сервера, эхо-сообщения между серверами 
4. Возможность запуска SCADA-системы как локального проекта на панельном ПК 

## 9. Безопасность и администрирование доступа

1. Авторизация, уровень доступа, администрирование, изменение прав доступа в режиме реального времени 
2. Поддержка блокировок ресурсов и ролей для управления критическими операциями.
3. Возможность блокировки работы пользователей администратором 
4. Ролевая модель доступа (RBAC) с настройкой прав на экраны, теги, операции
5. Поддержка интеграции с LDAP/Active Directory / SSO (SAML/OpenID Connect)
6. Аудит действий пользователей (who/what/when)
7. Подтверждение операций (двухэтапная авторизация для критичных действий)
8. Ограничение по временному окну и IP (по необходимости)


## 10. Жизненный цикл проекта и DevOps

1. Поддержка систем контроля версий проектов, откат к предыдущим версиям 
2. CI/CD, тестирование и быстрое развертывание проектов 
3. Экспорт-импорт проектов
